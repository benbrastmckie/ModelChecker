# Phase 2 Handoff: Extend Abundance Constraint with Performance Profiling

**Status**: COMPLETED
**Timestamp**: 2026-05-29

## What Was Done

Added three new abundance constraint methods to `semantic.py`:
1. `full_abundance_constraint` - ForAll/Exists with variable shift (strategy B)
2. `skolem_full_abundance_constraint` - Skolem function `shift_of(world, delta)` (strategy C)
3. `capped_skolem_abundance_constraint` - Skolem function with boundary caps (strategy D)

Also added helper `matching_states_when_shifted_var` for variable shift amounts.

Created profiling script at `code/src/model_checker/theory_lib/bimodal/profile_abundance.py`.

## Profiling Results

Key finding: With M=2, ALL abundance strategies still find countermodels to BM_TH_1.
This is a structural limitation: M=2 only has 2 valid intervals ([-1,0] and [0,1]) and
intervals can only shift by ±1, which is insufficient to cover all perpetuity shifts.

With M=3 and `capped_skolem_abundance_constraint`:
- BM_TH_1 (Box A -> Future A): timeout/unknown (no countermodel found) - CORRECT
- BM_TH_2 (Box A -> Past A): timeout/unknown (no countermodel found) - CORRECT
- Solve time: ~30s (hits max_time limit, which is acceptable for a theorem)

## Decision

Selected `capped_skolem_abundance_constraint` as the default strategy. Updated
`build_frame_constraints` to call `capped_skolem_abundance_constraint()` instead of
`skolem_abundance_constraint()`. BM_TH_1/BM_TH_2 settings will be updated to M=3 in Phase 3.

## Test Results

All 40 non-timeout bimodal tests still pass with the new capped Skolem abundance strategy.

## Deviations

- Plan mentioned a shift range of `{-(2*(M-1)), ..., 2*(M-1)}`. The capped constraint uses
  interval-boundary conditions directly rather than an explicit range variable. This is equivalent
  but more efficient since it avoids unnecessary quantification over shifts that would produce
  out-of-bounds intervals.
- `can_shift_forward`/`can_shift_backward` methods retained (not removed) -- they are still
  used by the legacy `build_abundance_constraint` and `skolem_abundance_constraint` methods
  which are kept for reference/fallback.
