\abstract{%
The development and comparative analysis of semantic theories---particularly hyperintensional and non-classical logics---has been hindered by the lack of systematic computational tools.
We present \modelchecker{}, a programmatic framework that treats semantic theories as executable programs, compiling truth conditions into SMT constraints for automated model discovery.
The theory-agnostic architecture enables rapid implementation of diverse semantic frameworks (classical, modal, temporal, truthmaker semantics) with systematic comparative analysis.
Constraint-based exploration of finite models provides evidence for validity by ruling out small countermodels, leveraging Z3 solver efficiency.
Computational tractability (solver performance, timeout rates, maximum model size) provides a rigorous, quantifiable measure of theoretical complexity, replacing subjective aesthetic evaluations with empirical metrics.
The extensible \theorylib{} library of semantic theories supports modular composition, sharing, and citation to advance semantic research.
We demonstrate implementation of four major semantic frameworks (\theorylogos{} bilateral truthmaker semantics with 19 operators, \theoryexclusion{} unilateral semantics, \theoryimposition{} counterfactual theory, \theorybimodal{} temporal-modal logic) with over 177 validated examples.
To our knowledge, this represents the first computational implementation of bilateral truthmaker semantics integrating hyperintensional operators (ground, essence, propositional identity), counterfactual conditionals, modal operators, and SMT-based finite model finding, demonstrating feasibility of programmatic semantic theory development at scale.
}

\section{Introduction}\label{sec:intro}

\subsection{The Challenge of Semantic Theory Development}\label{sec:challenge}

Semantic theories have traditionally been developed through hand-constructed models and manual proofs.
This approach carries significant limitations: difficulty verifying complex inference patterns across multiple operators, no systematic way to compare theories on identical test cases, barriers to entry for implementing emerging semantic frameworks, limited exploration of model space due to the burden of manual example construction, and validation bottlenecks where checking validity claims requires extensive manual computation prone to error.

Perhaps most significantly, theories are typically assessed via subjective aesthetic criteria---``elegance,'' ``simplicity,'' ``naturalness''---rather than empirical measures.
Debates about theoretical complexity remain unresolved due to the lack of objective metrics.
Different theorists may disagree profoundly about whether a theory is ``simple'' or ``elegant.''

\subsection{The Computational Turn in Semantics}\label{sec:computational-turn}

Recent work in computational metaphysics and automated reasoning has demonstrated the value of applying formal verification tools to philosophical logic~\cite{benzmuller2014,oppenheimer2011}.
These approaches have validated specific arguments and tested metaphysical theories, but have not provided general-purpose infrastructure for developing entire semantic frameworks.

\subsection{Our Contribution}\label{sec:contribution}

We present \modelchecker{}, a programmatic framework that:
\begin{enumerate}
    \item \textbf{Compiles semantic theories to SMT constraints}: Truth conditions become executable Z3 constraints.
    \item \textbf{Enables modular theory composition}: Operators loaded selectively with automatic dependency resolution.
    \item \textbf{Supports systematic comparison}: Run identical examples across multiple semantic frameworks.
    \item \textbf{Provides finite model exploration}: Constraint-based search through state spaces with configurable size.
    \item \textbf{Implements \theorylib{}}: Extensible library of semantic theories as reusable, citable components.
    \item \textbf{Establishes objective complexity measures}: Computational tractability (solver performance, timeout rates, feasible model sizes) replaces subjective aesthetic evaluations with empirical, quantifiable metrics.
\end{enumerate}

\section{The Programmatic Methodology}\label{sec:methodology}

\subsection{Three-Level Architecture}\label{sec:architecture}

Semantic theories function as pipelines from syntax to models, organized across three distinct levels.

\subsubsection{Level 1: Syntax Layer (Theory-Independent)}

The syntax layer handles formula parsing from \LaTeX{} notation to abstract syntax trees.
The parser recognizes structural patterns without knowing operator semantics; AST nodes map to semantic implementation classes.
This language-agnostic design means the same parser handles classical, modal, and hyperintensional operators.

\subsubsection{Level 2: Semantics Layer (Theory-Specific Plugin)}

Truth conditions generate Z3 formulas through an extensible evaluation point structure.
The \texttt{eval\_point} dictionary enables diverse semantic frameworks:
\begin{itemize}
    \item Modal: \texttt{\{"world": w\}}
    \item Bimodal: \texttt{\{"world": w, "time": t\}}
    \item Hyperintensional: Access to both worlds and partial states
\end{itemize}

Operator interface patterns vary by theory type.
Extensional theories implement \ztrueat{}$(w, s, \text{eval\_point})$.
Bilateral theories add \zfalsify{}, \zextverify{}, and \zextfalsify{}.
Frame constraints encode structural semantic principles (e.g., transitivity of accessibility relations).

\subsubsection{Level 3: Model Layer (Theory-Dependent Interpretation)}

The Z3 solver finds satisfying assignments or proves unsatisfiability.
Model extraction converts Z3's internal representation to semantic structures, and each theory defines how to interpret and display models.

\paragraph{Technical Example.}
Consider the formula $\nec(A \imp B)$ (necessity of conditional):

\begin{lstlisting}[caption={From formula to constraint}]
# Syntax Layer:
Parse: Necessity[Implication[A, B]]

# Semantics Layer (Modal):
# For all worlds w:
#   true_at(w, Box(A -> B)) <=> ForAll w'[R(w,w') -> true_at(w', A -> B)]

# Generates Z3 constraint:
ForAll([w_prime],
    Implies(R(w, w_prime),
        Implies(verify(w_prime, A),
                verify(w_prime, B))))

# Model Layer:
# Z3 finds: {w0, w1, w2}, R = {(w0,w1), (w1,w2)},
#           verify(w1, A) = True, verify(w1, B) = True, ...
\end{lstlisting}

\subsection{The Builder Pattern Orchestration}\label{sec:builder}

Three coordinating classes manage the complete pipeline.

\texttt{BuildModule} loads example files, manages theory selection, and coordinates multiple examples.
It takes as input a Python module with \texttt{premises}, \texttt{conclusions}, and \texttt{settings}, producing results across all examples, optionally in multiple theories.

\texttt{BuildExample} executes the complete pipeline for a single example with a specific theory.
The process includes: (1) parse formulas using the syntax layer, (2) instantiate theory-specific semantics, (3) generate Z3 constraints from semantic rules, (4) invoke solver with timeout, and (5) extract and format results.
A fresh Z3 context per example prevents constraint contamination.

\texttt{BuildProject} generates new theory implementations as complete Python packages, with scaffolding including \texttt{semantic.py}, \texttt{operators.py}, \texttt{examples.py}, tests, and documentation.

\subsection{Settings-Driven Semantic Control}\label{sec:settings}

The hierarchical configuration system (from highest to lowest priority) includes:
\begin{enumerate}
    \item Command-line flags: \texttt{--contingent}, \texttt{--maximize}, \texttt{--save}
    \item Example-specific settings: \texttt{settings = \{"N": 4, "iterate": 5\}}
    \item Module \texttt{general\_settings}: Apply to all examples in file
    \item Theory defaults: \texttt{default\_settings} in semantic class
    \item System defaults: Framework-level fallbacks
\end{enumerate}

\subsection{Constraint Generation Process}\label{sec:constraints}

Constraints fall into three categories:

\textbf{Frame Constraints} (3--5 per theory) define semantic structure independent of atomic propositions.
Examples include ``possible states closed downward'' and ``at least one world exists.''
These are loaded once per theory instantiation.

\textbf{Model Constraints} (15--60 per example) are applied per atomic proposition in formulas.
They control proposition behavior: fusion closure, classical constraints (\texttt{non\_empty}, \texttt{disjoint}).
The count scales with the number of sentence letters.

\textbf{Evaluation Constraints} (2--5 per example) encode specific example requirements: premises true at designated world(s), at least one conclusion false (for countermodel search).

A typical constraint count ranges from 20 to 130+ depending on formula complexity, number of atomic propositions, settings enabling optional constraints, and theory sophistication.

\subsection{Computational Complexity of Semantic Primitives and Arity}\label{sec:complexity}

The computational complexity of a semantic theory depends not only on how many Z3 primitives it declares, but critically on the \emph{arity} (number of arguments) of those primitives and how they interact in constraints.

\subsubsection{Objective Complexity Measurement}

Traditional semantic theory evaluation relies heavily on subjective aesthetic criteria: ``elegance,'' ``naturalness,'' ``parsimony,'' and ``intuitiveness.''
These evaluations are inherently subjective and contested.

Computational complexity provides a rigorous, objective alternative.
The tractability of a semantic theory---measured by solver performance, timeout rates, and maximum feasible model size---is an empirical fact, not an aesthetic judgment.
A theory that times out at $N=3$ is objectively more computationally complex than one tractable at $N=6$, regardless of how ``elegant'' its formulation appears.

Key advantages of computational measures include: (1) empirical verification where complexity claims can be tested and replicated, (2) quantitative comparison where theories are ranked by measurable performance metrics, (3) theory-independence where the same benchmarks apply across different semantic frameworks, (4) predictive power where performance characteristics guide practical applicability, and (5) optimization guidance where specific sources of complexity are identified for targeted improvement.

\subsubsection{Arity and Quantifier Complexity}

Z3 function declarations have varying arity, which directly impacts quantifier structure:

\begin{lstlisting}[caption={Primitive function arity}]
# Unary functions (1 argument)
possible = z3.Function('possible', BitVecSort(N), BoolSort)
# Constraint: ForAll s[possible(s) -> ...]  (1 quantifier)

# Binary functions (2 arguments)
verify = z3.Function('verify', BitVecSort(N), AtomSort, BoolSort)
# Constraint: ForAll s,A[verify(s,A) -> ...]  (2 quantifiers)

# Ternary functions (3 arguments)
selection = z3.Function('selection', BitVecSort(N), BitVecSort(N), AtomSort, BoolSort)
# Constraint: ForAll w,w',A[selection(w,w',A) -> ...]  (3 quantifiers)
\end{lstlisting}

Each additional argument introduces another quantifier in constraints, increasing solver difficulty exponentially.
Quantifier alternation is especially significant: $\forall s \forall A[\ldots]$ is relatively tractable, while $\forall A[\exists s[\ldots]]$ is much harder, and triple alternation $\forall w[\exists w'[\forall s[\ldots]]]$ is very difficult.

\subsubsection{Counterintuitive Result: Theoretical Simplicity vs.\ Computational Complexity}

The \theoryexclusion{} theory provides a striking illustration of how theoretical parsimony can diverge from computational tractability.
\theoryexclusion{} was explicitly motivated by the goal of reducing complexity through a unilateral approach to negation---dropping falsification entirely and using only a single primitive (\zwitness{}) plus an exclusion relation.
This appears simpler on paper: fewer base concepts, no bilateral evaluation.

However, \theoryexclusion{} proves far less computationally tractable than the bilateral \theorylogos{} theory.
Despite having a ``simpler'' theoretical foundation and eliminating the falsification primitive entirely, the \theoryexclusion{} theory requires a ternary \zexcludes{}$(s_1, s_2, A)$ primitive to capture negation.
This ternary primitive forces 3-quantifier constraints: $\forall s_1 \forall s_2 \forall A[\ldots]$.
The result: \theoryexclusion{} has 7.3$\times$ slower average solving time than bilateral \theorylogos{} (extensional).

This demonstrates that theoretical simplicity (fewer primitives, unilateral structure) does not guarantee computational simplicity.
The \emph{arity} of required primitives matters more than their count.

\subsubsection{Design Principle: Minimize Arity}

When implementing a new semantic theory, prefer binary over ternary (factor complex relations when possible), implicit over explicit (encode relationships as constraints rather than primitives), and stratified quantifiers (avoid alternation when possible).

The \theorylogos{} theory demonstrates this principle: fusion could be a ternary primitive \zfusion{}$(s_1, s_2, s_3)$, but instead uses bitvector OR ($s_1 \mathbin{|} s_2$), enabling $N=6$ whereas ternary fusion would limit to $N=4$.

\subsection{Complexity Classification}\label{sec:complexity-classification}

\subsubsection{Decision Problem Complexity}

The satisfiability problem for the modal fragment of \theorylogos{} inherits classical complexity bounds.

\begin{proposition}[Modal Fragment Complexity]
For the modal operators $\nec$ and $\poss$ over S4 frames:
\begin{enumerate}
    \item The satisfiability problem is PSPACE-complete~\cite{ladner1977}.
    \item The finite model property holds.
\end{enumerate}
\end{proposition}

The hyperintensional operators (ground, essence, propositional identity) operate over a finite state lattice, preserving decidability.
Recent work establishes that truthmaker logics on semilattices are decidable with the finite model property~\cite{knudstorp2023}.

\subsection{Encoding Correctness}\label{sec:encoding-correctness}

We establish that the Z3 encoding faithfully represents truthmaker semantics.

\begin{definition}[Encoding Function]\label{def:encoding}
Let $\mathcal{E}: \mathcal{L}_{\theorylogos} \times \statespace \to \text{Z3Formula}$ be the encoding function mapping formulas and states to Z3 constraints:
\begin{itemize}
    \item $\mathcal{E}(p, s) = \zverify(s, p)$ for atomic $p$
    \item $\mathcal{E}(\lneg A, s) = \zfalsify(s, A)$
    \item $\mathcal{E}(A \land B, s) = \mathcal{E}(A, s) \land \mathcal{E}(B, s)$
    \item $\mathcal{E}(A \lor B, s) = \exists s_1, s_2[s = \fuse{s_1}{s_2} \land \mathcal{E}(A, s_1) \land \mathcal{E}(B, s_2)]$
    \item $\mathcal{E}(\nec A, w) = \forall w'[\zaccessible(w,w') \imp \mathcal{E}(A, w')]$
    \item $\mathcal{E}(\poss A, w) = \exists w'[\zaccessible(w,w') \land \mathcal{E}(A, w')]$
\end{itemize}
The remaining operators (counterfactuals, constitutive operators) extend this mapping following their semantic clauses in Section~\ref{sec:logos}.
\end{definition}

\begin{theorem}[Encoding Soundness]\label{thm:soundness}
For any $\theorylogos$ formula $\varphi$ and finite state space $\statespace$ with $|\statespace| = 2^N$:
\[
\text{If } \mathcal{E}(\varphi) \text{ is UNSAT in Z3, then } \varphi \text{ has no } \theorylogos\text{-model over } \statespace.
\]
\end{theorem}

\begin{proof}[Proof sketch]
By structural induction on $\varphi$.

\textbf{Base case (atomic).}
$\mathcal{E}(p, s) = \zverify(s, p)$ is satisfiable iff there exists a valuation assigning $\zverify(s, p) = \top$.
Frame constraints ensure $s$ is possible and not null when $\zverify(s,p)$ holds.

\textbf{Inductive case (conjunction).}
By IH, $\mathcal{E}(A, s)$ is satisfiable iff $s \verifies A$ in some model.
$\mathcal{E}(A \land B, s) = \mathcal{E}(A, s) \land \mathcal{E}(B, s)$ is satisfiable iff both conjuncts are, iff $s$ verifies both $A$ and $B$.
By the truthmaker clause for conjunction (Definition~\ref{def:hyperintensional}), this holds iff $s \verifies A \land B$.

\textbf{Inductive case (necessity).}
$\mathcal{E}(\nec A, w) = \forall w'[\zaccessible(w,w') \imp \mathcal{E}(A, w')]$.
By IH, each $\mathcal{E}(A, w')$ correctly encodes $w' \models A$.
The universal quantification encodes ``for all accessible worlds,'' which is the standard Kripke semantics for necessity.

The remaining cases (negation, disjunction, modalities, counterfactuals, constitutive operators) follow similarly, with each encoding clause mirroring the corresponding semantic clause.
\end{proof}

\begin{theorem}[Countermodel Extraction]\label{thm:countermodel}
If $\mathcal{E}(\lneg \varphi)$ is SAT with model $M$, then $M$ can be extracted as a $\theorylogos$-countermodel to $\varphi$.
\end{theorem}

\begin{proof}[Proof sketch]
Z3's model extraction provides truth values for all $\zverify(s, p)$ and $\zfalsify(s, p)$.
These directly constitute the verification/falsification functions of the semantic model.
Frame constraints ensure the extracted model satisfies $\theorylogos$ structural requirements (downward closure, fusion closure, world existence).
\end{proof}

\begin{remark}[Limitations of Proof Sketch]
The encoding correctness argument above is a proof sketch, not a machine-checked proof.
Mechanized verification in a proof assistant (Coq, Isabelle/HOL, or Lean 4) remains future work.
The 177 validated examples provide extensive empirical evidence for correctness, and regression testing on every commit guards against encoding errors.
See Section~\ref{sec:limitations} for further discussion.
\end{remark}

\section{Finite Model Exploration and Countermodel Methodology}\label{sec:finite-model}

How can finite model checking provide evidence for semantic validity claims when model classes are infinite?

\subsection{The Finite Model Approach}\label{sec:finite-approach}

The rationale for finite model exploration rests on several observations.
First, countermodels disprove validity: finding one countermodel (of any size) proves invalidity.
Second, many logics have a bounded model property---if a countermodel exists, a small one exists.
Third, failure to find countermodels in progressively larger spaces increases confidence in validity.
Fourth, finite spaces enable SMT solver efficiency.

This is not a complete decision procedure.
The approach is sound: countermodel implies argument invalid.
However, completeness does not hold in general: no countermodel does not imply validity.
The exception is for logics with the finite model property (many modal logics), where search is complete up to the model size.

The trade-off sacrifices completeness for practical feasibility on complex theories (hyperintensional semantics), rapid prototyping and experimentation, and systematic exploration of model diversity.

\subsection{State Space Representation}\label{sec:state-space}

States are represented as $N$-bit vectors where each bit position corresponds to atomic ``stuff.''
For $N=3$:
\begin{align*}
000_2 = 0 &\mapsto \nullstate \text{ (null state)} \\
001_2 = 1 &\mapsto a \\
010_2 = 2 &\mapsto b \\
011_2 = 3 &\mapsto \fuse{a}{b} \text{ (fusion of $a$ and $b$)} \\
100_2 = 4 &\mapsto c \\
101_2 = 5 &\mapsto \fuse{a}{c} \\
110_2 = 6 &\mapsto \fuse{b}{c} \\
111_2 = 7 &\mapsto \fuse{a}{\fuse{b}{c}} \text{ (full state)}
\end{align*}

Total states: $2^N$ (8 states for $N=3$, 16 for $N=4$, etc.).

Mereological operations map to bitvector operations.
Fusion (state combination) uses bitwise OR: $\fuse{a}{c} = 001_2 \mathbin{|} 101_2 = 101_2$.
The part-of relation uses bitwise AND test: $a \parthood \fuse{a}{c}$ iff $(001_2 \mathbin{\&} 101_2 = 001_2)$, which is true.
Compatibility (overlap) uses bitwise AND non-zero: $a \cap b = 001_2 \mathbin{\&} 010_2 = 000_2$ implies incompatible, while $a \cap \fuse{a}{c} = 001_2 \mathbin{\&} 101_2 = 001_2$ implies compatible.

Z3's bitvector theory is highly optimized, mereological constraints compile to bit operations, and the representation scales ($N$ up to 64 feasible for simple theories).

\subsection{Constraint-Based Model Discovery}\label{sec:constraint-discovery}

The key insight is to let the solver find satisfying assignments rather than enumerating all $2^N$ states.

The process begins by declaring Z3 variables:
\begin{lstlisting}[caption={Z3 variable declarations}]
# State space
states = [z3.BitVec(f's{i}', N) for i in range(2**N)]

# Verification function
verify = z3.Function('verify', z3.BitVecSort(N), AtomSort, z3.BoolSort())

# Worlds (maximal consistent states)
worlds = [z3.BitVec(f'w{i}', N) for i in range(num_worlds)]
\end{lstlisting}

Constraints are then generated: frame constraints such as $\forall s[\zpossible(s) \imp \zpossible(\text{part}(s))]$ (downward closure), model constraints such as $\forall A[\text{contingent}(A) \imp (\exists s[\zverify(s,A)] \land \exists s'[\zfalsify(s',A)])]$, and evaluation constraints such as $\zverify(w_0, \text{premise}_1) \land \zverify(w_0, \text{premise}_2) \land \lneg \zverify(w_0, \text{conclusion})$.

The solver is then invoked, and if SAT, the model is extracted.

\begin{table}[htbp]
\centering
\caption{Constraint-Based vs.\ Enumeration Approach}
\label{tab:constraint-vs-enum}
\begin{tabular}{@{}lrrrr@{}}
\toprule
\textbf{Approach} & $N=3$ & $N=4$ & $N=5$ & $N=6$ \\
\midrule
Enumeration (states) & 8 & 16 & 32 & 64 \\
Models to check & 256 & 65K & 4B & Intractable \\
Constraint-based & $\sim$30 & $\sim$50 & $\sim$80 & $\sim$120 \\

The constraint-based approach scales better because the solver prunes the search space via unit propagation and conflict-driven clause learning.

\subsection{Evidence for Validity via Countermodel Exclusion}\label{sec:validity-evidence}

The progressive search strategy starts small ($N=3$) for fast search that rules out simple countermodels, increases gradually ($N=4, 5, 6$) for more complex models with longer search, and accumulates evidence where each UNSAT result strengthens the validity claim.

\subsection{Model Iteration for Diversity}\label{sec:iteration}

Setting \texttt{iterate: k} finds multiple distinct models:
\begin{lstlisting}[caption={Model iteration}]
settings = {'N': 4, 'iterate': 5}
\end{lstlisting}

The algorithm finds the first model $M_1$, adds a blocking constraint $\text{model} \neq M_1$, solves again to find $M_2$, adds $\text{model} \neq M_1 \land \text{model} \neq M_2$, and repeats until $k$ models are found or UNSAT.

Difference tracking is theory-specific: \theorylogos{} tracks different verification/falsification patterns, world structures, and part-of relations; \theoryexclusion{} tracks different witness assignments and exclusion relations; \theorybimodal{} tracks different temporal orderings and modal accessibility patterns.

Applications include model space exploration, robustness testing, theory refinement (identifying unintended models suggesting missing constraints), and educational use.
Typically 5--10 iterations are feasible before timeout.

\subsection{Limitations and Future Work}\label{sec:finite-limitations}

Current limitations include incompleteness (cannot prove validity in general), state space ceiling ($N > 6$ often intractable), timeout sensitivity for complex formulas, and inability to handle infinite models.

Mitigation strategies include symmetry breaking (add constraints reducing isomorphic models such as ordering worlds by state number), incremental solving (reuse constraint bases across examples), and parallel search (explore different $N$ values concurrently).

Future extensions include bounded model checking to prove completeness for specific logic fragments, abstraction refinement to iteratively refine model size, quantified Boolean formulas to encode some infinite properties finitely, and integration with proof assistants to use finite models to guide Coq/Isabelle proofs.

\section{Case Study: Logos Theory and Unified Hyperintensional Semantics}\label{sec:logos}

The \theorylogos{} theory demonstrates the framework's capacity to implement sophisticated semantic theories---a unified approach to hyperintensional reasoning needed for the language of thought.

\subsection{Philosophical Background: The Language of Thought}\label{sec:philosophical-background}

The core idea of truthmaker semantics~\cite{fine2017tms} is that propositions are characterized by what makes them true or false.

\begin{definition}[Hyperintensional Operator]\label{def:hyperintensional}
An operator $\mathcal{O}$ is \emph{hyperintensional} if there exist formulas $A$ and $B$ such that:
\begin{enumerate}
    \item $A$ and $B$ are necessarily equivalent: $\nec(A \leftrightarrow B)$, yet
    \item $\mathcal{O}(A)$ and $\mathcal{O}(B)$ may differ in truth-value.
\end{enumerate}
Equivalently, substitution of necessary equivalents within the scope of $\mathcal{O}$ does not preserve truth.
\end{definition}

\begin{remark}[Hyperintensional Operators in \theorylogos{}]\label{rem:hyperintensional-operators}
The following \theorylogos{} operators are hyperintensional:
\begin{itemize}
    \item \textbf{Conjunction} ($\land$): $A \land B$ and $B \land A$ are necessarily equivalent but may have distinct verifiers, hence $\lneg((A \land B) \propident (B \land A))$ is satisfiable.
    \item \textbf{Propositional identity} ($\propident$): $A \propident B$ requires identical verifiers \emph{and} falsifiers, not merely necessary equivalence.
    \item \textbf{Ground} ($\ground$): $A \ground B$ requires every verifier of $B$ to contain a verifier of $A$ as a part.
    \item \textbf{Relevant implication} ($\relimpl$): Requires subject-matter overlap beyond material truth.
\end{itemize}
These distinctions arise from bilateral evaluation: propositions are characterized by their exact verifiers and falsifiers, not merely their truth-values across possible worlds.
\end{remark}

\textbf{Bilateral Evaluation.}
Verifiers are states that make a proposition true.
Falsifiers are states that make a proposition false.
Gaps are partial states that do neither.

For proposition $A$: ``It is raining,'' verifiers include states where rain occurs ($r$, $\fuse{r}{w}$, $\fuse{r}{c}$, \ldots), falsifiers include states where no-rain obtains ($\lneg r$, $\fuse{\lneg r}{c}$, \ldots), and gaps are partial states indeterminate about rain.

\textbf{Mereological Structure.}
States form a partial order under part-of ($\parthood$).
The null state ($\nullstate$) is part of everything and verifies/falsifies nothing.
Atomic states are minimal non-null states ($a, b, c, \ldots$).
Fusions combine states via the $\sqcup$ operation.
Closure properties (whether $s \verifies A$ and $s \parthood t$ implies $t \verifies A$) are theory-dependent.

\subsection{Logos Implementation Strategy}\label{sec:logos-implementation}

Design goals include a comprehensive operator suite (covering extensional, modal, constitutive, counterfactual, and relevance operators), modular architecture (operators organized by semantic function, selectively loadable), empirical validation (extensive example suite testing operator interactions), and performance optimization (constraints optimized for tractability despite complexity).

\textbf{Z3 Primitive Declarations:}
\begin{lstlisting}[caption={\theorylogos{} Z3 primitives}]
class LogosSemantics(SemanticDefaults):
    def declare_z3_primitives(self):
        # Core evaluation functions
        self.verify = z3.Function('verify', BitVecSort(N), AtomSort, BoolSort)
        self.falsify = z3.Function('falsify', BitVecSort(N), AtomSort, BoolSort)

        # Mereology
        self.possible = z3.Function('possible', BitVecSort(N), BoolSort)

        # Modal accessibility
        self.accessible = z3.Function('accessible', BitVecSort(N), BitVecSort(N), BoolSort)

        # Worlds (maximal consistent states)
        self.is_world = z3.Function('is_world', BitVecSort(N), BoolSort)
\end{lstlisting}

\textbf{Frame Constraints} (9 constraints): (1) null state exists: $\zpossible(\nullstate)$; (2) downward closure: $\forall s[\zpossible(s) \imp \forall t[t \parthood s \imp \zpossible(t)]]$; (3) fusion closure: $\forall s,t[\zpossible(s) \land \zpossible(t) \imp \zpossible(\fuse{s}{t})]$; (4) world existence: $\exists w[\zisworld(w)]$; (5) world maximality: worlds contain all possible states as parts; (6) modal reflexivity: accessibility relation reflexive (for S4/S5); (7) modal transitivity: accessibility relation transitive; (8) no null verification: $\forall A[\lneg \zverify(\nullstate, A) \land \lneg \zfalsify(\nullstate, A)]$; (9) exclusive verification: $\forall A,s[\lneg(\zverify(s,A) \land \zfalsify(s,A))]$ (optional via settings).

\subsection{Operator Semantics: Five Subtheories}\label{sec:operator-semantics}

\subsubsection{Extensional Operators}

\textbf{Conjunction} ($\land$):
\begin{lstlisting}[caption={Conjunction operator}]
class Conjunction(OperatorDefaults):
    def extended_verify(self, state, sentence, eval_point):
        A, B = sentence.sentences
        return z3.And(
            self.semantics.extended_verify(state, A, eval_point),
            self.semantics.extended_verify(state, B, eval_point)
        )

    def extended_falsify(self, state, sentence, eval_point):
        A, B = sentence.sentences
        return z3.Or(
            self.semantics.extended_falsify(state, A, eval_point),
            self.semantics.extended_falsify(state, B, eval_point)
        )
\end{lstlisting}

Interpretation: $s$ verifies $A \land B$ iff $s$ verifies both $A$ and $B$; $s$ falsifies $A \land B$ iff $s$ falsifies $A$ or falsifies $B$.

\textbf{Negation} ($\lneg$): Negation swaps verifiers and falsifiers.
The verify method returns \texttt{extended\_falsify(state, A)}, and falsify returns \texttt{extended\_verify(state, A)}.

Other extensional operators: disjunction defined via De Morgan ($\lneg(\lneg A \land \lneg B)$), implication as $\lneg A \lor B$, biconditional as $(A \imp B) \land (B \imp A)$, top ($\top$) verified by all states, bottom ($\bot$) falsified by all states.

\subsubsection{Modal Operators}

\textbf{Necessity} ($\nec$):
\begin{lstlisting}[caption={Necessity operator}]
class Necessity(OperatorDefaults):
    def true_at(self, world, sentence, eval_point):
        A = sentence.sentences[0]
        accessible_worlds = [w for w in self.semantics.worlds
                            if self.semantics.accessible(world, w)]
        return z3.And([
            self.semantics.true_at(w_prime, A, eval_point)
            for w_prime in accessible_worlds
        ])
\end{lstlisting}

Interpretation: $\nec A$ true at $w$ iff $A$ true at all accessible worlds from $w$.
Possibility ($\poss$) is defined as $\lneg \nec \lneg A$.
The accessibility relation is configurable (reflexive, transitive, symmetric), determining the modal logic (S4, S5, K, etc.).

\subsubsection{Constitutive Operators}

\textbf{Ground} ($\ground$): $A \ground B$ (``$A$ grounds $B$'') holds when $B$'s verifiers are among $A$'s verifiers.
\textbf{Essence} ($\essence$): Similar to ground but for essential connections---necessary grounding.
\textbf{Propositional Identity} ($\propident$): Strongest equivalence---exactly same verifiers and falsifiers.

\subsubsection{Counterfactual Operators}

\textbf{Counterfactual Conditional} ($\boxright$): $A \boxright B$ true at $w$ iff in all alternative $A$-worlds to $w$, $B$ is true.
Evaluation occurs at alternative possible worlds where the antecedent is true, requiring a similarity ordering on worlds.

\textbf{Might Counterfactual} ($\diamondright$): Dual---``If $A$ were true, $B$ might be true'' (some close $A$-world has $B$ true).

\subsubsection{Relevance Operators}

\textbf{Relevant Implication} ($\relimpl$): Requires actual connection between $A$ and $B$, not just material implication.
If antecedent verified, consequent must be verified with mereological connection.
Additionally requires subject matter overlap between $A$ and $B$.

\subsection{Unification Across Domains}\label{sec:unification}

\theorylogos{} provides a single semantic framework for classical logic ($\lneg, \lor, \land, \imp, \leftrightarrow, \top$), modal logic ($\nec, \poss$), counterfactual reasoning ($\boxright, \diamondright$), content semantics ($\oplus, \propident$), grounding theory ($\mathbin{|}, \ground$), and relevance logic ($\relimpl$).

The key insight is that a unified language of thought requires operators from all these domains.
Cognitive agents reason modally (what's possible), counterfactually (what would happen if\ldots), content-sensitively (what thoughts are about), and explanatorily (why things are true).
This unification demonstrates the framework's expressiveness.

\subsection{Empirical Validation and Test Suite}\label{sec:validation}

The 177 examples span multiple categories.

\textbf{Extensional} (42 examples): classical tautologies ($A \imp A$, $A \lor \lneg A$), conjunction properties (commutativity, associativity), implication (modus ponens, hypothetical syllogism).

\textbf{Modal} (38 examples): S4 axioms ($\nec A \imp A$, $\nec A \imp \nec \nec A$), possibility ($\poss A \leftrightarrow \lneg \nec \lneg A$), interaction ($\nec(A \imp B), \nec A \vdash \nec B$ K axiom), novel hyperintensional ($\nec A \land \nec B \imp \nec(A \land B)$ in some models).

\textbf{Constitutive} (35 examples): ground transitivity ($A \ground B, B \ground C \vdash A \ground C$), identity symmetry ($A \propident B \imp B \propident A$), distinguishing ($A \leftrightarrow B \not\Rightarrow A \propident B$).

\textbf{Counterfactual} (32 examples): modus ponens ($A, A \boxright B \vdash B$), contraposition failures, strengthening antecedent failures, transitivity failures (as expected theoretically).

The validation process involves manual construction (theory expert constructs example and predicts validity/invalidity), automated testing (each example runs in $<5$ seconds at $N=3$ or $N=4$), regression testing (all 177 examples run on every commit), and cross-theory validation (some examples run across multiple theories to confirm divergent predictions are intentional).

\subsection{Qualitative Validation: Case Studies}\label{sec:case-studies}

\paragraph{Case Study: Counterfactual Non-Transitivity.}
Does $A \boxright B, B \boxright C$ imply $A \boxright C$?
Classical conditionals: Yes (hypothetical syllogism).
Counterfactual logic: No.

\begin{lstlisting}[caption={Testing counterfactual transitivity}]
example = {
    'premises': [
        'A \\boxright B',  # If A, then B
        'B \\boxright C'   # If B, then C
    ],
    'conclusions': ['A \\boxright C'],  # If A, then C?
    'settings': {'N': 4}
}
result = run_example(example, logos_counterfactual)
\end{lstlisting}

Result: Invalid.
Countermodel with worlds $\{w_0, w_1, w_2, w_3\}$ shows alternative $A$-worlds differ from alternative $B$-worlds.
At $w_0$: $A \boxright B$ true (alternative $A$-world $w_1$ has $B$ true); $B \boxright C$ true (alternative $B$-world $w_2$ has $C$ true); $A \boxright C$ false (alternative $A$-world $w_1$ has $C$ false).
This confirms Lewis's philosophical analysis computationally~\cite{lewis1973}.

\subsection{Validation Against Philosophical Literature}\label{sec:literature-validation}

Comparison of \theorylogos{} against Fine's imposition semantics reveals structural equivalence (both satisfy frame constraints) but logical independence (neither implies the other).
Key differences include definability of necessity: $\nec A \define (\top \boxright A)$ is invalid in imposition semantics but valid in \theorylogos{} for $N \leq 6$.

The ``jump problem'' in imposition semantics allows imposing the null state $\nullstate$ on world $a$ to yield unrelated world $b$, violating minimal change intuition.
\theorylogos{}'s constructive approach avoids this.

\section{Related Work}\label{sec:related}

\subsection{Automated Reasoning in Philosophy}\label{sec:reasoning-philosophy}

\textbf{Computational Metaphysics.}
Benzmüller et al.\ used Isabelle/HOL and LEO-II for automated verification of Gödel's ontological argument~\cite{benzmuller2014}.
Similarity: applying automated reasoning to philosophical logic.
Difference: focus on specific argument validation, not general semantic framework.
Our contribution: framework for developing entire semantic theories, not just verifying arguments.

Oppenheimer \& Zalta used Prover9/Mace4 for testing metaphysical theories~\cite{oppenheimer2011}.
Similarity: using theorem provers/model finders for philosophy.
Difference: first-order logic focus, less emphasis on modularity.
Our contribution: SMT-based approach with richer semantic structures (states, fusion, bilateral evaluation).

\textbf{Automated Theorem Proving for Modal Logic.}
Fitting developed systematic tableaux procedures for modal logics~\cite{fitting1983}.
Similarity: automated validation of modal inferences.
Difference: proof search focus, not model construction.
Our contribution: model-finding emphasis with Z3 enables countermodel inspection.

\textbf{Modal Logic Solvers.}
MLSolver~\cite{friedmann2010} addresses the satisfiability problem for modal fixpoint logics including the mu-calculus, CTL*, and PDL using parity game reduction.
PRONOM~\cite{pronom2021} is the first theorem prover for the non-normal modal logic cube, extending coverage beyond normal modal logics.
These tools focus on validity/satisfiability decision procedures using possible-worlds semantics rather than model construction with truthmaker semantics.
Our contribution differs in: (1) model-finding emphasis (exploring actual countermodels rather than just deciding satisfiability), (2) bilateral evaluation structure (explicit verifiers and falsifiers), and (3) hyperintensional operators beyond modal logic.

\subsection{SMT Solvers in Verification}\label{sec:smt-verification}

De Moura \& Bjørner's Z3~\cite{demoura2008} provides efficient SMT solving with theories (arithmetic, bitvectors, arrays).
We apply Z3 to philosophical semantics rather than software verification.
Barrett et al.'s survey of SMT techniques~\cite{barrett2009} describes potential alternative backends (CVC4/CVC5).

Clarke et al.\ on model checking~\cite{clarke1999} covers automated verification of finite-state systems.
Similarity: finite model exploration.
Difference: hardware/software focus, temporal properties.
Our contribution: semantic theories as systems to verify.

\subsection{Semantic Frameworks and Implementations}\label{sec:semantic-frameworks}

\textbf{Proof Assistants.}
Coq~\cite{bertot2004}, Isabelle/HOL~\cite{nipkow2002}, and Lean provide interactive theorem proving with rich type theories.
Similarity: formal verification of logical systems.
Difference: interactive proof construction, high learning curve.
Our contribution: automated model finding, lower barrier to entry, rapid prototyping.

\textbf{Truthmaker Semantics.}
Fine's bilateral truthmaker framework~\cite{fine2017tms,fine2017content} introduces exact verification and falsification as primitive notions, providing the philosophical foundation for hyperintensional semantics.
Subsequent theoretical developments include Yablo's work on aboutness~\cite{yablo2014} and Jago's applications to grounding and content~\cite{jago2020}.
Knudstorp~\cite{knudstorp2023} establishes decidability and finite model property results for truthmaker logics on semilattices.
These remain primarily theoretical; our implementation translates Fine's informal semantics into executable Z3 constraints.

\textbf{Hyperintensional Logic.}
Transparent Intensional Logic (TIL)~\cite{til2010} provides a hyperintensional framework for natural language analysis, using procedural semantics based on typed lambda calculus.
Recent work investigates hyperlogic with structured propositions and counterfactual conditionals.
These frameworks focus on theoretical foundations and proof theory rather than automated model finding.
Our contribution: executable implementation enabling empirical exploration of hyperintensional theories with automated countermodel discovery.

\section{Discussion and Future Directions}\label{sec:discussion}

\subsection{Contributions Summary}\label{sec:contributions}

\textbf{Theoretical Contributions.}
(1) Programmatic methodology for semantics: treating semantic theories as executable programs where truth conditions compile to SMT constraints.
(2) Comprehensive bilateral truthmaker implementation: to our knowledge, the first computational realization of Fine's informal semantics, formalized with 19 operators across 5 semantic domains.
(3) Theory-agnostic framework design: clean separation between core infrastructure and theory-specific logic with zero coupling between theories.
(4) Finite model methodology: constraint-based approach to finite model exploration with progressive search strategy.

\textbf{Practical Contributions.}
(1) \theorylib{}: reusable semantic components with 4 implemented theories and modular operators with automatic dependency resolution.
(2) Systematic comparative analysis: side-by-side theory comparison on identical examples with database of comparative predictions.
(3) Educational resources: interactive Jupyter notebooks with comprehensive documentation.
(4) Open source infrastructure: MIT licensed, public repository with CI/CD.

\subsection{Limitations and Threats to Validity}\label{sec:limitations}

\subsubsection{Internal Validity: Encoding Correctness}

\textbf{Semantic encoding fidelity.}
The Z3 encoding must faithfully represent truthmaker semantic clauses.
We mitigate this threat through: (1) extensive test suite coverage (177 validated examples), (2) regression testing on every commit, and (3) comparison with hand-constructed examples from the truthmaker semantics literature~\cite{fine2017tms}.
However, the encoding has not been mechanically verified in a proof assistant (see Section~\ref{sec:encoding-correctness}).

\textbf{Frame constraint soundness.}
Frame constraints require expert judgment to ensure they correctly capture semantic principles.
Incorrect constraints could admit spurious models or exclude valid ones.
Mitigation: constraints are derived directly from published definitions in~\cite{fine2017tms,fine2017content}.

\textbf{BitVector representation limits.}
States are represented as $N$-bit bitvectors.
For $N > 64$, this exceeds native word size.
Current implementation is limited to $N \leq 16$, corresponding to $\leq 65{,}536$ states.

\subsubsection{External Validity: Generalizability}

\textbf{Finite model restriction.}
The framework can only explore finite models.
Logics without the finite model property (e.g., certain second-order extensions) cannot be completely characterized.
Mitigation: many philosophically significant modal and hyperintensional logics have the finite model property~\cite{knudstorp2023}.

\textbf{Theory coverage.}
Results generalize to theories expressible within our framework (bilateral semantics, mereological state spaces).
Theories requiring different primitives (e.g., neighborhood semantics for non-normal modal logics) would require framework extension.

\textbf{Example selection bias.}
The 177 examples were constructed to test specific operator behaviors, not sampled randomly.
They may not represent ``typical'' formulas users will test.

\subsubsection{Construct Validity: Complexity Measurement}

\textbf{Solver time as complexity proxy.}
We use Z3 solving time as a measure of theory complexity.
This conflates semantic complexity with encoding efficiency and solver heuristics.
Mitigation: we also report constraint counts and timeout rates.

\textbf{Feasible model size threshold.}
Defining tractability as ``$N \leq 6$ before timeout'' is arbitrary.
Different applications may have different size requirements.

\subsubsection{Scalability Limitations}

\textbf{Exponential state space.}
The state space grows as $2^N$, limiting practical exploration.
At $N=6$ (64 states), many formulas time out; $N=7$ (128 states) is often infeasible.
See Section~\ref{sec:finite-limitations} for detailed discussion.

\textbf{Constraint explosion.}
Complex formulas with many atomic propositions generate thousands of constraints.
The ternary/quaternary primitives required by some operators (e.g., \zcloser{} for counterfactuals) exacerbate this.

\textbf{Quantifier alternation.}
Nested modal operators introduce quantifier alternation, which is exponentially harder for SMT solvers.
Formulas like $\nec\poss\nec p$ may time out even at $N=3$.

\subsubsection{Limitations Not Addressed}

\textbf{Infinite models.}
No support for reasoning about infinite model classes.

\textbf{Soundness proof automation.}
The encoding correctness argument (Section~\ref{sec:encoding-correctness}) is a proof sketch, not a machine-checked proof.
Mechanized verification in Coq, Isabelle/HOL, or Lean 4 remains future work.

\textbf{Multi-theory comparison validity.}
When comparing theories on the same example, differences may reflect encoding choices rather than deep semantic differences.

\subsection{Broader Impact}\label{sec:impact}

\textbf{Philosophy and Logic.}
Computational turn in semantics: shift from informal to executable semantic theories with empirical validation as standard practice.
Cumulative knowledge building: theories as reusable, citable components.
Lower barrier to entry: non-specialists can implement theories.

\textbf{Computer Science and AI.}
Formal reasoning systems: hyperintensional semantics for content-sensitive AI, planning systems with counterfactual reasoning.
Knowledge representation: richer semantic structures for ontologies.
Program verification: SMT techniques applied to semantic theories.

\textbf{Cognitive Science.}
Language of thought: computational models of mental representation with content-sensitive reasoning.
Belief attribution: modeling hyperintensional belief states.

\subsection{Conclusion}\label{sec:conclusion}

We have presented \modelchecker{}, a programmatic framework for developing modular semantic theories using SMT solvers.
The framework enables rapid development (theory-agnostic architecture allows implementing new semantic frameworks in days/weeks), supports systematic comparison (run identical examples across multiple theories), provides empirical validation (extensive test suites ensure correctness), facilitates finite model exploration (constraint-based approach efficiently searches model spaces), implements rich semantics (bilateral truthmaker semantics with integrated hyperintensional operators), promotes modularity (operators, theories, and tools as reusable components), and lowers barriers (accessible Python API and comprehensive documentation).

The \theorylib{} vision provides an extensible library of executable semantic theories serving as a comparative research platform, archival and citation system, educational resource, and collaborative community hub.

The \theorylogos{} case study demonstrates the framework's capacity to implement sophisticated theories with 19 operators across 5 semantic domains, unified hyperintensional semantics for language of thought, and 177 validated examples confirming correctness.

By treating semantic theories as executable programs, we enable a more rigorous, empirical, and collaborative approach to developing the formal languages needed for philosophical inquiry, cognitive science, and AI.
The future of semantic theory is computational, modular, and collaborative---and it starts with frameworks like \modelchecker{} making that future accessible to researchers today.
